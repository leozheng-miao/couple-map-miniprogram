- 项目名称：情侣地图微信小程序

- 交付日期：2026-05-29

- 代码仓库：`leozheng-miao/couple-map-miniprogram`

## 产品定位

情侣地图是一款面向双人关系的私密回忆地图小程序。用户可以创建情侣空间，通过邀请码邀请另一位用户加入，然后共同记录去过的餐厅、酒店、景点和其他地点。每个地点支持照片、评分、小记和便利贴，相册页会聚合空间内所有地点照片。

`v1.0.0` 的目标是完成一个可上线、可双人共享、可持续记录的最小完整版本。

## 已交付功能

### 情侣空间

- 创建情侣空间。
- 通过邀请码加入空间。
- 刷新邀请码。
- 获取当前用户所在空间。
- 双人共享同一空间内的地点、照片和便利贴。

### 地图与地点

- 地图首页展示已记录地点。
- 按地点类型筛选地图记录。
- 新增地点。
- 编辑地点。
- 删除地点。
- 地点类型支持餐厅、酒店、景点、其他。
- 地点支持名称、地址、坐标、去过日期、评分和小记。

### 搜索与选点

- 支持腾讯位置服务 WebService API 进行 POI 搜索。
- 未配置地图 Key 或搜索不可用时，支持手动地图选点。
- 本版本不读取用户实时位置。

### 照片与相册

- 地点支持上传多张照片。
- 第一张照片可作为地点封面。
- 地图卡片、地点详情、相册页均支持图片展示。
- 双方成员可以互相查看对方上传的照片。
- 跨用户图片展示通过云函数生成临时访问 URL 完成。

### 便利贴

- 地点详情中支持新增便利贴。
- 便利贴支持文字和图片。
- 便利贴支持多种颜色。
- 支持编辑和删除便利贴。

### 数据导出

- 通过云函数导出空间内地点和便利贴数据。
- 当前版本用于用户备份和开发排查，不提供复杂后台管理界面。

## 未纳入 v1.0.0 的范围

- 多空间切换。
- 公开内容流。
- 好友体系。
- 评论、点赞、收藏等社交功能。
- 复杂行程规划。
- 后台管理系统。
- 商业化能力。
- 实时定位、持续定位、位置变更监听。

## 技术架构

### 小程序端

- 微信原生小程序。
- TypeScript 编写页面、组件和工具函数。
- WXML/WXSS 实现手账风视觉界面。
- 所有业务数据读写通过云函数完成。

### 云开发端

- 微信云开发。
- 云数据库保存用户、空间、地点和便利贴。
- 云存储保存用户上传的照片。
- 云函数负责身份识别、空间成员校验、业务读写和图片临时 URL 转换。

### 第三方服务

- 腾讯位置服务 WebService API。
- 仅用于 POI 搜索，不用于实时定位。

## 云数据库集合

需要创建以下集合：

- `users`
- `spaces`
- `places`
- `notes`
- `checkins`
- `app_config`

建议权限：

```json
{
  "read": false,
  "write": false
}
```

所有核心读写由云函数完成，云函数会校验当前用户是否属于目标空间。

## 云存储策略

- 用户上传的地点照片和便利贴照片保存到云存储。
- 数据库中保存原始 `fileID`。
- 前端展示时不直接依赖跨用户读取 `fileID`。
- `listPlaces`、`getPlaceDetail`、`getAlbum` 会在成员鉴权后调用 `cloud.getTempFileURL()`，返回临时 URL 给前端展示。

## 权限与审核状态

本版本不申请以下接口：

- `getLocation`
- `onLocationChange`
- `startLocationUpdate`

小程序不读取用户实时位置。地点坐标来自用户主动搜索或手动点选。

微信隐私保护指引应按实际能力填写：

- 图片选择。
- 图片上传。
- 云存储保存用户主动上传的照片。
- 不填写定位类接口使用场景。

## 发布流程摘要

1. 在微信开发者工具打开项目根目录。
2. 确认小程序 AppID 和云环境 ID。
3. 创建云数据库集合。
4. 选择云函数根目录对应云环境。
5. 逐个部署业务云函数，选择“创建并部署：云端安装依赖”。
6. 配置腾讯位置服务 Key，并开启 WebService API。
7. 编译小程序。
8. 真机测试核心路径。
9. 上传代码。
10. 提交微信审核。

## v1.0.0 验收重点

- 用户 A 创建空间。
- 用户 B 通过邀请码加入空间。
- 用户 A 新增地点并上传照片。
- 用户 B 可以看到用户 A 的地点和照片。
- 用户 B 新增便利贴或照片。
- 用户 A 可以看到用户 B 的便利贴或照片。
- 相册页能聚合双方上传的地点照片。
- 删除地点或便利贴后，列表和详情不再展示对应内容。
- 上传审核不再触发定位接口审核问题。
- 真机调试包体不超过微信限制。

## 已知注意点

- `miniprogram/env.ts` 包含本地云环境 ID 和腾讯位置服务 Key，提交远端时应避免提交真实私密配置。
- `project.private.config.json` 是开发者工具本地配置，不应作为版本能力依据。
- 顶层 `cloudfunctions/common` 只是共享源码模板；部署时部署具体业务函数目录。
- 腾讯位置服务 Key 只用于云函数 `searchPoi` 调用 WebService API。
- 如果图片跨用户不可见，优先检查 `listPlaces`、`getPlaceDetail`、`getAlbum` 是否已重新部署。

## 下一版本建议

### v1.1.0 优先级建议

- 多空间支持：允许用户创建或加入多个空间，并在个人中心切换。
- 地点时间线：围绕地点形成多次打卡记录。
- 图片管理：支持删除单张照片、设置封面、图片排序。
- 更完整的数据导出：导出结构化 JSON 或生成可分享纪念页面。
- 空间成员信息：展示双方昵称、头像、加入时间。
- 审核与隐私文档自动检查：提交前检查定位接口、隐私声明和包体资源。

### 技术债建议

- 将云函数共享 `common` 的同步复制做成脚本，避免新增公共模块后漏复制到业务函数。
- 为云函数增加轻量单元测试或本地调用脚本。
- 为小程序端补充关键页面的冒烟测试清单。
- 将腾讯位置服务 Key 改为云函数环境变量或云端配置，减少客户端配置暴露。
